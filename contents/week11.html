Pre-Tutorial Learning Activities

Get design tips

If you haven’t already, Also look at CS2103 Project: Points to Ponder for some design tips useful when refining the product design.

Use other models [Individual, 2pts]

Do one of the following (try to ensure at least one member does each):
  • Relate (no need to draw) at least 2 “non-examinable” UML models to your project.
    Explain why/where the model is potentially applicable or why it is not applicable to the project.
  • Draw an activity diagram to represent a workflow/algorithm in your part of the project.
    If your part of the project does not have any workflows, you can draw an activity diagram to represent the workflow of a user using your team’s product.
    Bring the diagram to the tutorial.
  • Draw an OODM for your project. Remember, OODM is about the problem domain, not about the solution.
    Classes such as GUI, Manager, Storage should not appear in the OODM but it can have classes such as Task and Reminder.
    It should be about how a person deals with his/her schedules and tasks. Bring the diagram to the tutorial.

Some integration/system testing [Team, 5pts]

Ensure you have *some* automated integration and/or system tests in place.
i.e. testing beyond individual unit testing and covering a major part of the system, not just one method/class.
e.g. a test that takes the full user command as the input and verify that it is executed properly.

Reach V0.3 [Team, 5pts]

V0.3 is noticeably better than the previous version, and preferably, ready for dog fooding.
The code should be tagged as V0.3 in the repo.
There should be at least 20 realistic tasks in the demo database covering a wide variety of tasks.
That is, each team member should try to enter 20+ realistic items into the database using the product.
Even better if you use the product to track your own schedule for real (this is called dog fooding).
ALL members should have a demo-ready V0.3 version in their computer when you come to the tutorial.
Any of you should be able to do the demo at a moment’s notice. This demo to be done by someone who did not demo the product in previous milestone demos.
No debugging/integration allowed prior to the demo.

Contributed to V0.3 [Individual, 5pts]

[3 pts] Committed code for this version.
[1 pt] Has a demo-ready version of V0.3 in own computer and ready to do a demo.
[1 pt] Has 20+ tasks in the demo database.
