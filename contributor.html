Contributor guide

General

  • HTML can be used freely in any part of the page.
  • Please follow the class/id names and hierarchies strictly because the JS functions and CSS rules target very specific class/id names and hierarchies.
  • Try not to use multibyte characters (e.g ‘, ’, “, ”).
  • Images shall be hosted ourselves except for those taken from external website (with source acknowledged). Also, they shall be lossless-ly compressed via this service.
  • Some frequently-used classes have been defined for paragraph texts. Currently there are:
    • highlighted
    • highlighted-darktext, can be used if the highlighted text has bright colour, in which the previous class will do this: highlighted
    • important
    • small-font
    • keyword
    • additional-info. This is a class for div wrapper.
    Feel free to define new styles. Place them in appropriate section in common.css.
  • Samples can be seen in various pages in contents/ folder.

Contributing to schedule

Follow the template in templates/weeklytemplate.html.

  • The sub-panels can be copied for as many as needed.
  • Replace * with the week number and % with week number - 1.
  • Please do not change the class names to allow JS functions and CSS rules to work.
  • If there are links to external files for download (e.g lecture notes, slides, ...), place them in files/ folder for better organization.

Contributing to handbook

Contribute section contents as individual HTML files. Save as contents/handbook-*.html, where * is the keyword of the section. Please choose one that's representative.

To make the section appear in the actual handbook (handbook.html), add these lines:

<h2>The section name</h2> // use h3 for sub-sections
<div id="handbook-*"> // * is again the same keyword used
</div>
Then, on the script at the bottom of the page, add the keyword to the array variable sections.

Embedding parts of handbook to the schedule

To make a certain keyword/keyphrase trigger an opening of an embed handbook, follow this template:

<span onclick="getHandbookSection('*', '**');"><u>Some keyword</u></span>
// * = the same keyword of the section used above
// ** = the selector for the embedded window

The embedded window should be a classless div wrapper with a unique id.

Testing new week entries

Name the file week*.html and place it in contents/ folder.

Right after the last existing week entry on schedule.html, add these lines:

<div class="weeklyschedule">
     <h3 id="header-content-week*" class="preloaded">Week *</h3>
     <div id="content-week*">
     </div>
</div>

Load index.html and subsequently schedule.html from the sidebar, then open the newly added accordion. Check manually if everything is fine.
